# PLATFORM RULES — non-negotiable operational rules (Henna + Feather)

Violating these loses work, pay, or platform standing regardless of annotation quality.

1. **Two-platform mirror**: every action taken in Feather must be mirrored on the
   Annotation Platform (Henna). Submit⇔Submit, Release⇔Release. Never cross.
2. **Attempt URL**: after claiming in Feather, copy the browser URL (the task ID CHANGES
   after claiming) and paste it into Henna's Attempt URL field. Wrong/pre-claim URL breaks
   the link between your work and the task.
3. **Never** click `Cancel task` or `Escalate issue` in Feather. Task problems ⇒
   `Release task` + topic in #general with the annotation link.
4. Task not Unclaimed in Feather when opened ⇒ touch nothing in Feather, answer **No** on
   Henna's "could you submit?" question, Release on Henna, report in #general.
5. **Claim timer = 4× AHT.** Exceeding it can unassign the task: work lost, unpaid.
   Don't claim without enough uninterrupted time.
6. **Don't let tasks time out** — timeouts and admin releases count against the daily
   limit; voluntary releases don't.
7. Release is for tasks you genuinely cannot do (domain/language/access), not for hard or
   slow ones. Release patterns are monitored.
8. Submission is final until review — verify before submitting, not after.
9. Use the full time allocation. Suspiciously fast submissions get extra scrutiny.
10. **Fix flow (Needs work)**: open the task via the Attempt URL in Task History (NOT the
    original Task Variables link) → read ALL reviewer comments → `Make edits` → fix EVERY
    listed issue → `Mark as fixed`. Fixing only some issues guarantees another SBQ.
11. Same email everywhere (platform registrations, LinkedIn, Feather login via LinkedIn).
12. Announcement channels are read-only for annotators; questions go to #general as threads.
